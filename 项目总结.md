## 🎯 核心优化策略

### 1. 顺序模态训练 (Sequential Training)

**原理**：不同时计算三个模态，而是依次处理

```
原方案流程（同时）:
├─ FDG前向  ─┐
├─ AV45前向 ─┤── 同时占用显存
└─ TAU前向  ─┘
└─ 一次性反向传播
   └─ 更新参数

优化方案流程（顺序）:
├─ FDG: 前向 → 反向 → 释放 ✓
├─ AV45: 前向 → 反向 → 释放 ✓
└─ TAU: 前向 → 反向 → 释放 ✓
   └─ 更新参数
```

**效果**：显存降低 **60-70%**

### 2. 梯度累积 (Gradient Accumulation)

**原理**：分多步累积梯度，模拟更大batch size

```python
# accumulation_steps = 2
# 相当于 batch_size = 2，但显存只需要 bs=1 的一半

步骤1: loss1.backward()  # 梯度累积
步骤2: loss2.backward()  # 梯度累积
步骤3: optimizer.step()  # 更新参数
步骤4: optimizer.zero_grad()
```

**效果**：显存进一步降低 **50%**，训练更稳定

### 3. 混合精度训练 (Mixed Precision)

**原理**：使用FP16进行计算，FP32保存权重

```python
with autocast('cuda'):  # 自动FP16
    output = model(input)
    loss = criterion(output, target)

scaler.scale(loss).backward()  # 缩放防止下溢
scaler.step(optimizer)
```

**效果**：显存降低 **40-50%**，速度提升 **10-20%**

## 🔧 故障排除

### 问题1: 仍然显存溢出

**解决方案**（按顺序尝试）：

1. 增加梯度累积：`accumulation_steps = 4`
2. 减小输入尺寸：`[160,192,160] → [128,128,128]`
3. 减少模型通道数：`num_channels=(16,32,32,64)`

### 问题2: 损失不收敛

**检查项**：

1. 损失是否正确缩放：`loss / accumulation_steps`
2. 学习率是否过大：降至 `1e-5`
3. 混合精度问题：临时关闭 `use_amp = False`

### 问题3: 多GPU速度未提升

**可能原因**：

1. 数据加载瓶颈 → 增加 `num_workers`
2. 通信开销过大 → 减少GPU数量
3. batch太小 → 增加 `accumulation_steps`

## 🎓 理论基础

### 为什么顺序计算能节省显存？

```
同时计算需要的显存:
= 模型参数 + 3×(输入+输出+中间激活值)
≈ 2GB + 3×(8GB) = 26GB

顺序计算需要的显存:
= 模型参数 + 1×(输入+输出+中间激活值)
≈ 2GB + 1×(8GB) = 10GB

节省: 16GB (62%)
```

### 梯度累积的数学原理

```
标准训练（bs=4）:
Loss = (L1 + L2 + L3 + L4) / 4
∇W = ∂Loss/∂W

梯度累积（bs=1×4步）:
步骤1: ∇W += ∂L1/∂W / 4
步骤2: ∇W += ∂L2/∂W / 4
步骤3: ∇W += ∂L3/∂W / 4
步骤4: ∇W += ∂L4/∂W / 4
       W = W - lr × ∇W

结果: 完全等价！
```
