# replicaLT

A two-stage MRI-to-PET generation framework with plasma-guided conditioning.

> - **`plasma_train.py` is the main method of the project**
> - **root `train_legacy.py` is only the legacy / baseline implementation** and should not be treated as the primary entrypoint for the current method

---

## 1. Project overview

`replicaLT` is a **two-stage 3D MRI-to-PET generation framework**.

The overall design is:

1. **Stage A: representation alignment** (`adapter_v2/train.py`)
   - Learn reusable multimodal alignment across TAU image tokens, MRI tokens, diagnosis labels, and plasma biomarker values.
   - Produce the semantic basis for later plasma-conditioned generation via a CoCoOp-style model (`CoCoOpTAUModel`).
   - Currently configured with a single plasma biomarker key (`pT217_F`) by default; up to 5 keys are supported (see `adapter_v2/config.yaml`).

2. **Stage B: plasma-guided PET generation** (`plasma_train.py`)
   - Load precomputed `plasma_emb` for each subject (generated by `precompute_plasma_emb.py`).
   - Use `plasma_emb` as Token 0 and a tracer-specific BiomedCLIP text feature as Token 1.
   - Train a **3D MRI-to-TAU PET generator** using a DDPM scheduler with velocity-field (`v_prediction`) targets, following a rectified-flow-style linear interpolation.
   - **Current active configuration: TAU-only** (FDG and AV45 loss weights are set to zero).

This repository should therefore **not** be interpreted as a single end-to-end script. The alignment stage and the generation stage are deliberately decoupled.

## 2. Main entrypoints in this branch

### Current project mainline
- **`plasma_train.py`**

This is the main training script for the current method.

Its role is to:
- load precomputed `plasma_emb`
- combine plasma embeddings with modality-specific text features
- build a fixed `(B, 2, 512)` conditioning tensor
- perform **3D MRI -> PET** generation

### Upstream alignment stage
- **`adapter_v2/train.py`**

This is the main entrypoint for the representation/alignment stage.

Its role is to:
- train multimodal alignment using TAU, MRI, diagnosis, and plasma information
- provide the semantic basis used to derive plasma-guided conditioning in Stage B

### Legacy / comparison-only code
- **`train_legacy.py`**

This script is **not** the primary method in the current branch.

It should be treated as:
- a **legacy implementation**
- a **baseline / comparison method**
- a historical reference for earlier conditioning strategies

When discussing the current project method, always start from **`plasma_train.py`**, not from `train_legacy.py`.

## 3. Stage A: alignment training (`adapter_v2/train.py`)

### Purpose
Stage A learns a reusable multimodal alignment model across:
- TAU image patch tokens (vision cache `.vision.pt`)
- MRI image patch tokens (vision cache `.mri_vision.pt`)
- Diagnosis labels / class prompts (CN / MCI / AD)
- Plasma biomarker values

The goal of this stage is **not** final PET synthesis. Its purpose is to build a strong semantic representation layer, especially for **plasma-related conditioning**.

### Core code path
When analyzing Stage A, prioritize the following files:

1. `adapter_v2/train.py`
2. `adapter_v2/dataset.py`
3. `adapter_v2/models.py`
4. `adapter_v2/losses.py`
5. `adapter_v2/precompute_cache.py`
6. `adapter_v2/config.yaml`

### Data and caching
- Vision caches (`.vision.pt` / `.mri_vision.pt`) are auto-generated before training via `precompute_cache.py`.
- Samples come from `TAUPlasmaDataset`; subject-level grouping is enforced via `SubjectBatchSampler` to avoid false negatives.
- Train/val split is saved to `adapter_v2/fixed_split.json` (subjects-based, compatible with `run_probe_and_retrieval.py`).

### Plasma biomarker configuration
Five biomarker keys are available: `AB42_AB40_F`, `pT217_F`, `pT217_AB42_F`, `NfL_Q`, `GFAP_Q`.  
**Current default in `config.yaml`:** `selected_keys: ["pT217_F"]` (single key).  
Each key has a dedicated semantic prompt defined in `plasma.prompts_by_key`.  
Inter-sample normalization defaults to `minmax`; intra-sample weighting defaults to `sigmoid`.

### Model (`CoCoOpTAUModel`)
- **TAU branch:** patch tokens → `TokenAttentionPool` → `g_tau` (512-d)
- **MRI branch:** patch tokens → `TokenAttentionPool` → `g_mri` → `ContextNet` → dynamic context tokens `T_ctx` (ctx_len × 768)
- **Class branch:** class prompts + `T_ctx` → `BiomedCLIPTextEncoder` → class embedding (512-d)
- **Plasma branch:** plasma prompts + `T_ctx` → `BiomedCLIPTextEncoder` → per-key features; weighted sum via normalized plasma values → plasma embedding (512-d)
- All three embeddings are projected to a shared 512-d CLIP space via `ProjectionHead`.

### Loss (`compute_total_loss` in `losses.py`)
- `L_img_class`: image ↔ class prototype contrastive loss (CLIP-style InfoNCE)
- `L_img_plasma`: image ↔ plasma contrastive loss
- `L_class_plasma`: class ↔ plasma contrastive loss (lower weight)
- `L_reg`: high-similarity penalty or weak orthogonality regularization to prevent embedding collapse

### Validation metrics
1. **Linear probe:** Balanced Accuracy / Macro-F1 on image embeddings (LogisticRegression on val set)
2. **Context-injected classification:** Balanced Accuracy / Macro-F1 using class-prototype logits
3. **Plasma shuffle counterfactual:** within-class and cross-class plasma permutation tests; reports cosine score drop and margin drop to diagnose whether plasma encodes class-aligned vs. class-independent information

### Run artifacts
- Run directory: `adapter_v2/runs/<timestamp_pid>/`
- Best checkpoint: `ckpt/best.pt`
- Reproducibility metadata: `meta/repro_manifest.json` (includes `resolved_config.yaml`, `cli_args.json`, `runtime.json`, `requirements.txt`, git diffs)

## 4. Stage B: plasma-guided generation (`plasma_train.py`)

### Purpose
This is the **main method** of the current branch.

Its objective is to synthesize TAU PET from MRI while using plasma-derived semantic conditioning.

### Key idea
The generation model does **not** use plasma as a supervised regression target.

Instead, plasma is injected as two **conditioning tokens**:

- **Token 0:** precomputed `plasma_emb` (512-d, from `CoCoOpTAUModel` Stage A)
- **Token 1:** BiomedCLIP-encoded tracer-specific text feature (512-d)

These two tokens form a conditioning tensor of shape **`(B, 2, 512)`**, injected via cross-attention. This design requires **zero changes** to the core UNet architecture.

### Model architecture
`DistributedDiffusionModelUNet` (MONAI generative):
- `spatial_dims=3`, 1 input / 1 output channel
- `num_channels=(32, 64, 64, 128)`, `num_res_blocks=1`
- `attention_levels=(False, False, False, True)` — cross-attention only at the deepest level
- `cross_attention_dim=512`, `with_conditioning=True`, Flash Attention enabled
- Supports **1–4 GPU model parallelism** via `PLASMA_TRAIN_GPUS` environment variable

### Training formulation
Follows a **rectified-flow-style velocity-field regression** using a DDPM scheduler:

- Interpolated input: `x_t = t · PET + (1 − t) · MRI`, where `t = timestep / 1000`
- Velocity target: `v = PET − MRI`
- Loss: `MSE(v_pred, v)`
- Scheduler: `DDPMScheduler(prediction_type="v_prediction", num_train_timesteps=1000, schedule="scaled_linear_beta")`

**Phase transition at epoch ≥ 140:** `time_embedding` is fixed to `0`, shifting training toward deterministic single-step generation.

AMP mixed precision with `GradScaler` and Adam optimizer (lr = 2.5 × 10⁻⁵).

### Current tracer configuration (TAU-only)
Loss weights in `plasma_train.py`:
- `alpha = 0.0` — FDG disabled
- `beta  = 0.0` — AV45 disabled
- `gamma = 1.0` — TAU active

The codebase retains all multi-tracer infrastructure (modality texts, index transforms, per-tracer validation), but **only TAU steps contribute gradient updates** in the current configuration.

### Validation (every `val_interval` epochs)
Single-step integration (`N_sample=1`). Metrics per active tracer:
- **MSE** — used for best-checkpoint selection
- **SSIM** — 3D, `kernel_size=7`, `data_range=1.0`
- **PSNR** — `data_range=1.0`
- **MAE** — mean absolute error

All metrics are logged to TensorBoard. Axial / coronal / sagittal slice visualizations are also logged.

### Outputs
- TensorBoard logs: `runs/plasma_<MM.DD>_<pid>/`
- Checkpoints: `checkpoint/first_part_<epoch>.pth`
- Hyperparameters: `runs/<run_name>/hparams.json`

## 5. Precomputed plasma embeddings (`precompute_plasma_emb.py`)

### Required preprocessing
Before running `plasma_train.py`, run `precompute_plasma_emb.py` to generate per-subject embedding caches.

### Signal flow (per subject)
1. Load MRI vision cache → `mri_tokens` (196, 768)
2. Match plasma CSV → `plasma_values` + `plasma_mask`
3. Forward through **frozen** `CoCoOpTAUModel`:
   - `mri_tokens` → `TokenAttentionPool` → `g_mri` (512-d)
   - `g_mri` → `ContextNet` → `T_ctx` (ctx_len × 768)
   - `base_ctx_plasma + T_ctx` → `BiomedCLIPTextEncoder` → per-key features (K, 512)
   - weighted sum → `ProjectionHead` → `plasma_emb` (512-d)
4. Save: `{ptid}_plasma_emb.pt` → `{"plasma_emb": (512,), "plasma_mask": (K,)}`

Subjects missing a cache file fall back to a zero vector at training time (degrades to unconditioned generation).

## 6. Tracer strategy

### Framework capability
The codebase supports independent targets for FDG, AV45, and TAU. Each has:
- a dedicated modality-optimized text prompt encoded by BiomedCLIP
- a separate `index_transform` that injects the correct `(plasma_emb, modality_text)` pair as `context`
- per-tracer validation metrics

### Current active configuration
**TAU-only** (`alpha=0, beta=0, gamma=1` in `plasma_train.py`).

All FDG / AV45 infrastructure remains in the codebase for future multi-tracer experiments, but no gradient flows through those branches currently.
